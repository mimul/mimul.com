# 블로그 글쓰기 스타일

## 언어 및 문체

- 한국어로 작성
- 개발 블로그 스타일 유지 (기술적이고 명확하게)
- 불필요한 장황함 제거, 핵심 위주
- "~것이다" 반복, "덧붙여서" 남용 등 단조로운 문체 지양
- 번역투 표현 지양 ("~되고 있다" → "~됩니다", "어느 것도" → "모든")

## 문서 형식

- 코드 블록은 언어 반드시 명시 (```python, ```bash, ```yaml 등)
- 이미지는 해당 슬러그 디렉토리에 저장하고 상대 경로로 참조 (`./image.png`)
- 표(table)는 비교 항목이 3개 이상일 때 사용
- 긴 URL은 텍스트 링크로 표시 (`[링크명](URL)`)
- **볼드** 텍스트는 앞뒤에 공백 필수 — `공백**텍스트**공백` (MDX 렌더링 오류 방지)

## 프론트매터 규칙

- `title`: 30~60자, 독자가 얻는 것을 명확히 표현, **콜론 `:` 사용 금지** (Gatsby 빌드 오류 원인)
- `description`: 80~160자, SEO 키워드 포함, 독자 행동 유도형
- `tags`: 소문자 + 공백은 하이픈(`-`)으로 (`distributed-file-system`)
- `date`: 수정하지 않음
- `lastModificationTime`: 글 수정 시 오늘 날짜로 업데이트

## 구조 원칙

- 도입부에 대상 독자와 글에서 얻을 수 있는 것 명시
- 섹션 제목은 반복 없이 계층 구조로 구성
- 글 말미에 요약 또는 다음 글 예고로 마무리

## 글 품질 체크리스트

- **맞춤법**: "어플리케이션→애플리케이션", "두번째→두 번째", "첫째→첫 번째" 등 자주 틀리는 표현 확인
- **중복·과잉 출력 제거**: 코드 실행 결과는 핵심 필드만 발췌, 동일 내용 반복 섹션 압축
- **자기 참조 금지**: 시리즈 링크 목록에 현재 글 자신은 포함하지 않음
- **흐름**: 섹션 간 전환 문장으로 맥락 연결, 논리적 점프 없게 구성
